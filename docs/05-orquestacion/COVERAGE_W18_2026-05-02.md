## 5. Decisions

### W18-D-COV-1 · Merge wave-17-coverage-and-w16-tests INTO main alongside wave-17-bridge-helper-and-ad-platforms

The two W17 branches are independently complete and don't conflict (different files), but their value materializes only when both land. **Recommendation:** prepare a single integration PR `wave-17-merged` that fast-forwards both into `origin/main`, OR merge them into `wave-18-validation-and-coverage` as the integration target. Without this, every future coverage measurement on a base derived from one of the two will show false negatives on the files served by the other.

**Owner:** integration step before any W19 PRs.
**Effort:** ~10 min (no merge conflicts expected — disjoint file sets).

### W18-D-COV-2 · W19 prioritizes `mc-sync/route.ts` as the single biggest gap

At 46.77% stmt / 28.3% branch, `mc-sync` is twice as deep a gap as anything else in the tree. It's also operationally important — every learning cycle and HITL approval round-trip touches it. **Recommendation:** allocate the first half of W19 to a dedicated `mc-sync-coverage` task targeting ≥80% stmt / ≥70% branch.

**Owner:** TBD.
**Effort:** ~20-25 tests, est. 3-4 hours.

### W18-D-COV-3 · DEFER the ad-platform `fallback_mode` branch coverage

The 75% branch number on the 8 ad-platform endpoints looks like a gap but is a measurement artifact: the `withFallback` ternary is never exercised because the inner stub is synchronous and infallible. To cover it requires either (a) refactoring routes to inject the stub function (8 endpoints to touch, surface API change), or (b) using vi.mock to swap module bindings (fragile across the [platform] dynamic route). **Recommendation:** accept 75% as the steady-state for these endpoints. The fallback shape is identical across all 8 and is functionally tested via the bridge-fallback unit tests.

**Owner:** N/A (decision is to NOT do).

### W18-D-COV-4 · Add the missing tier-1 invariant test to bridge-fallback.ts (line 110-112)

The `if (!result)` branch in `withSupabaseResult` is uncovered. One 4-line test (`expect(withSupabaseResult(async () => null))`...) closes it. Trivial gap-fill, can ride a W19 PR.

**Owner:** trivial; first W19 PR to touch the file.
**Effort:** 5 minutes.
